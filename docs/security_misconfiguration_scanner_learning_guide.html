Security Misconfiguration Scanner API - Beginner Learning Guide

This printable guide explains the concepts used in this FastAPI project: APIs, routes, schemas, services, dependency injection, exception handling, server logging, testing, and deployment basics.

Current project status: the API validates scan requests and returns mock scan findings. It does not yet perform real HTTP security scanning. Unexpected errors are logged on the server and hidden from the client.

1. What This App Does

The app receives a target URL, validates it, passes it into a scan service, and returns a scan response. Today the findings are mock data.

POST /api/v1/scans
Content-Type: application/json

{
  "target_url": "https://example.com",
  "include_exposed_files": true
}

2. Main Request Flow

Client
  -> app/main.py creates FastAPI app
  -> app/api/routes/scans.py receives request
  -> app/schemas/scans.py validates body
  -> app/api/dependencies.py provides ScanService
  -> app/services/scans.py creates mock scan
  -> app/schemas/scans.py shapes response
  -> Client receives JSON

4. Why The Layers Exist

Routes should stay small. They should receive HTTP data, call services, and return responses. Services should contain business rules. Schemas should define data contracts. Core should contain app-wide infrastructure.

6. Important Pydantic Concepts

class ScanCreateRequest(BaseModel):
    model_config = ConfigDict(extra="forbid")

    target_url: HttpUrl = Field(...)
    include_exposed_files: bool = Field(default=True)

HttpUrl validates URLs. extra="forbid" rejects unknown fields. Field(default=True) gives a default value.

7. Exception Handling

Expected errors return useful JSON. Unexpected errors return safe generic JSON and are logged on the server.

ErrorStatusResponse Code
Invalid body422validation_error
Missing scan404not_found
Business rejection400bad_request
Unexpected crash500internal_server_error

8. Stack Trace Leakage Prevention

async def unhandled_exception_handler(request: Request, exc: Exception) -> JSONResponse:
    logger.exception(
        "Unhandled exception while processing %s %s",
        request.method,
        request.url.path,
    )
    return JSONResponse(
        status_code=status.HTTP_500_INTERNAL_SERVER_ERROR,
        content=ErrorResponse(
            error="internal_server_error",
            detail="An unexpected server error occurred.",
        ).model_dump(),
    )

The client receives a safe message. The server log receives the traceback. This is the correct production security pattern.

9. Testing

Tests use pytest and httpx.ASGITransport to call the app directly without starting a real server.

.venv/bin/python -m pytest -q

Dependency overrides let tests replace the real scan service with fake services that return success, raise 404, raise 400, or crash.
